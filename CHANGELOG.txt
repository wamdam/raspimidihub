RaspiMIDIHub Changelog

User-visible changes. Short tagged one-liners, no code or commit
hashes. Tags: Added, Improved, Fix, Changed, Removed.


2026-05-04 — Version 3.0.5

The arpeggiator just grew up. Three changes that turn a static
note-cycler into a live-programmable performance instrument:

Arpeggiator
- Added: Sustain pedal (CC 64) is now a temporary Hold. Press the
  pedal and the keys you release stay arping; lift it and only the
  notes you're physically holding right now survive. Stack a chord,
  pedal down, release the keys, jam new notes on top — when you
  lift the pedal, only what's currently under your fingers keeps
  playing. CC 64 respects the existing Arp Ch filter and is
  consumed (not passed through). Same effect for every pattern
  mode.
- Added: New notes play on the very next step. Hold C E G in `up`
  pattern, mid-cycle press D — D fires on the next tick and the
  arp continues from there, instead of starting over from the
  bottom. Pattern-aware seek so up / down / up-down / as-played all
  behave naturally.
- Added: New `programmed` pattern — live step-sequencer mode. Each
  keypress writes the next-to-fire step slot; multiple presses
  between ticks fan into consecutive slots (chord-spread); slots
  persist while any key is held or sustain is down, and clear when
  everything's released. The StepEditor renders the loaded note
  name (C4, F#3, …) under each step square. Disabled steps are
  skipped on write so a muted step never silently swallows an
  assignment. Multi-octave plays the slot sequence at +12 / +24 /
  …  on successive cycles.

Together: hold a chord and the sustain pedal, release the chord,
keep playing — sustained notes loop. Punch new notes in `programmed`
mode and watch them overwrite slots in real time as the cycle
catches up. Lift everything, the phrase ends. Press a single key,
a new one starts. Live performance, no preset switching.

- Improved: Drop button fade now interpolates XY pad cells too —

2026-05-04 — Version 3.0.3

- Fix: Update flow steps the system clock from an HTTP Date header
  before any HTTPS request. The Pi has no RTC and our system-prepare
  disables NTP, so on a cold boot the clock is stuck at last-shutdown
  — TLS to api.github.com then failed with "certificate not yet
  valid". HEAD against a plain-HTTP host (no TLS) reads the Date
  header and steps CLOCK_REALTIME if drift > 60 s. Both the cabled
  and transient-WiFi paths sync before probing.


2026-05-04 — Version 3.0.2

- Fix: MIDI Learn on the Add Mapping panel — was waiting silently
  forever because its EventSource never subscribed to `midi-activity`
  under the per-view SSE subscription model. Now subscribes once the
  connection event arrives, same shape as the NoteSelect Learn
  button. Note / CC capture works again.


2026-05-03 — Version 3.0.1

Major release. Bluetooth MIDI, Controllers (Mixer 8 / FX 6 /
Performance 16 / XY 4), drop buttons with bar-quantised fire, ALSA
queue scheduling for sample-accurate plugin output, USB tethering,
in-app updates with apt-resolved dependencies, per-device clock
vetoes, hostapd via systemd, SysEx Sender plugin.

Numbered 3.0.1 (skipping 3.0.0) because Debian's version comparator
treats `3.0.0a2-1` as **greater** than `3.0.0-1` — without an
explicit `~` separator the alpha trailer counts as a newer suffix.
Anyone testing the 3.0.0a2 alpha would have seen apt refuse the
upgrade to 3.0.0 as a downgrade.

Bluetooth MIDI
- Added: BLE-MIDI bridge — paired devices appear in the matrix with a
  Bluetooth icon (separate row + column for IN / OUT). End-to-end
  message coverage: notes, CC, program change, pitch bend, channel /
  poly aftertouch, plus MIDI Clock, Start, Stop, Continue and Song
  Position. SysEx pending.
- Added: Add Device → Bluetooth section — Scan, Connect, Disconnect,
  Forget. MIDI-only filter on by default; "Show all (N other)"
  reveals every BLE peripheral in range.
- Added: Just-Works pairing agent registered automatically. WIDI
  Master and similar peripherals that demand encrypted GATT subscribe
  pair without user interaction.
- Added: Auto-reconnect on service restart for paired BLE-MIDI
  devices.
- Added: Reconnect entry in the matrix context menu for paired-but-
  offline BLE-MIDI devices, alongside Forget.
- Added: BlueZ disconnect detection — when a peripheral goes out of
  range or is powered off, the matrix flips its row/column to offline
  within ~1-2 seconds.
- Added: /var/lib/bluetooth persistence under read-only root — tmpfs
  at boot restored from a tarball on /boot/firmware, snapshot taken
  on every change via inotify so bonds survive immediate power-off
  without a graceful shutdown.
- Changed: BlueZ's built-in `midi` plugin is disabled (systemd drop-
  in, `bluetoothd -P midi`) because it doesn't actually forward GATT
  notifications. We bridge BLE-MIDI in Python instead.

Controllers
- Added: Controller tab in the top nav — fullscreen play surface, swipe / arrow / dropdown to switch between instances, last-viewed remembered
- Added: Mixer 8 — 24 knobs / 8 faders / 16 buttons (CC 16-63 ch 1)
- Added: FX 6 — 18 knobs / 6 faders / 6 buttons (CC 16-45 ch 1)
- Added: Performance 16 — 16 macro knobs + 4 scene buttons (CC 16-35 ch 1)
- Added: XY 4 — 2 XY pads + 8 knobs + 4 buttons (CC 16-31 ch 1)
- Added: 4 drop buttons per controller — long-press to capture, tap to fire. Modes: Now / Bar / 2-Bar / 4-Bar / 8-Bar / 16-Bar, musical-grid quantised
- Added: Drop button — Sync-to-bars toggle, Fade-on-fire toggle, MIDI-note trigger with Learn, dual-slot scheduling (one fade + one hard drop side by side)
- Added: Drop button progress ring — segmented arc shows position in the cycle, peach pulse while scheduled, freezes on MIDI Stop
- Added: Per-cell rename, Ch + CC rebind, MIDI Learn (per axis on XY pads), and on/off CC values for buttons
- Added: 8 dark themes per controller (Default / Navy / Forest / Wine / Plum / Teal / Sienna / Slate)
- Added: XY pad spring — per-cell force + home (Bottom-left / Center); the dot returns to home after release at the chosen speed
- Added: Plugin Config in the device-detail panel renders a flat per-cell list with mini wheels for Ch / CC / On / Off. Maximize button jumps to the full Controller page
- Added: Pencil icon on the Controller bar opens the controller's plugin config directly
- Improved: Drop fire is sample-accurate — snapshot CCs land just ahead of the bar boundary, before any beat-1 event at the destination synth
- Changed: Drop button labels shortened: "Sync to bars" → "Sync", "Trigger Note" → "Trg. Note"

Routing & workflow
- Added: Single-tap context menu on every matrix cell, plugin / hardware header and mapping row
- Added: Connection clipboard — copy filter + mappings, paste into another cell
- Added: Plugin clipboard — copy a plugin instance, Paste-as-new clones it
- Added: Mapping clipboard — pastes as-is, or auto-bumps to a free slot on duplicate-conflict
- Added: URL routing — back / forward and bookmarks now work for /routing, /controller, /presets, /settings, and the open device panel
- Added: Edit menu item on hardware headers — opens the device-detail panel for MIDI monitor + test sender
- Added: Bottom-nav Routing icon shows a dark-red asterisk when in-memory state diverges from the saved config
- Improved: Disconnect sends a clean Note Off so notes don't stick
- Improved: Load Config only reapplies what changed; unrelated routing keeps flowing
- Improved: Multi-tab — several phones / browsers can be open at once without one being pushed out
- Improved: MIDI monitor on a destination device (e.g. Mixer 8) shows incoming events from upstream devices
- Improved: Bottom-nav Routing asterisk skips Controller live-play state — fader / knob / XY moves and drop-button fires don't dirty the config; only cell rename, rebind, drop button settings, captured snapshots, theme do
- Added: Per-device clock veto — every hardware device's detail panel has a "Drive system clock" toggle; off = engine drops Clock / Start / Stop / Continue from that source before they reach the bus, so multiple clock sources can be tamed without unticking Clock/RT in every connection's filter
- Added: Matrix clock indicator — 3-state (sole green / multi orange / blocked dim grey) so a configured-out clock source is still visible but obviously not driving the bus
- Fix: Engine re-subscribes the monitor port after CLIENT_EXIT/CLIENT_START hotplug — devices that re-enumerated with the same client id (typical for Elektron over USB) silently disappeared from the rate meter, clock indicator, and MIDI activity bar; audio path was unaffected because direct kernel subscriptions stayed up
- Changed: All matrix interactions go through the menu (no more dual tap-or-long-press)

Plugins
- Added: SysEx Sender — pick a .syx file in the device-detail panel, bytes stream straight to the connected destination (256-byte chunks, ~5 ms gap; nothing saved)
- Added: Clock Divider — emit one MIDI Clock for every N received (2..32)
- Added: UI Demo — showcase of every UI control type, makes no MIDI
- Added: Arpeggiator Trigger Note — toggle in the Pattern group with a Base wheel (MIDI Learn). Notes from Base..Base+14 set the Rate radio live without being arpeggiated. Use to remap rate from a hardware controller key
- Added: Arpeggiator Arp Ch / Ctrl Ch — channel filters on the input. Default Any. Set Arp Ch = 1 + Ctrl Ch = 16 to wire a separate keyboard / footswitch on ch 16 that only flips Rate, never plays
- Improved: Master Clock — clock ticks pre-scheduled through the kernel queue, rock-solid 24-PPQN under load
- Improved: MIDI Delay — every echo pre-scheduled at note-on time. Delay (ms) is now a Fader; sync rate covers 4 bars down to 1/16 with triplets
- Improved: Arpeggiator — gate parameter actually controls staccato/legato now (was effectively 100 % before)
- Improved: CC LFO — slow clock-sync rates added (up to 8 bars)
- Improved: Panic — momentary trigger button; second tap upgrades to "All Sound Off"

WiFi, updates & networking
- Added: USB tethering — plug a phone into a USB-A port, Settings shows the tethered URL as a clickable link
- Added: WiFi mode preference — radio between AP only / WiFi for updates / WiFi always
- Added: Software Updates card — auto-downloads newer GitHub releases, keeps the latest 3, installs with one tap
- Added: Software Updates "Include alpha / beta releases" toggle — default off (stable channel)
- Added: Rollback insurance — the running version's deb is always downloaded if not on disk and protected from prune, so after an upgrade you can always one-tap-revert
- Added: Update watchdog — forces the Pi back to AP mode if the orchestrator hangs in client mode
- Added: Update progress UI — live status, "we're alive" hopping dot, "reconnect to AP" help after 90 s of failed polls
- Added: Captive-portal probe access log + hostapd syslog routing — every /generate_204 etc. hit logs path + client IP + response time; hostapd associate / deauth events land in the journal under raspimidihub-hostapd.service so phone-disconnect post-mortems are possible
- Improved: "Reload App" + "stale, reload" reliably picks up the new build on phones (was getting stuck on Safari bf-cache)
- Changed: Install uses `apt install` (not `dpkg -i`), so upgrades that introduce new transitive packages pull them automatically when the Pi is online. The orchestrator inspects the deb's Depends and, if any are missing, routes the install through the same transient WiFi-switch dance used by Check-for-updates so it can never fail with a half-configured package. Offline downgrades to a stored version with already-satisfied deps still skip the dance and install fully offline.
- Changed: Captive-portal endpoints now serve a tiny static landing with a single "Open routing matrix" link instead of suppressing the OS captive flow. Captive webview stays inert (no JS, no SSE) — the user taps the link to open the SPA in their main browser
- Changed: Settings WiFi card consolidated — one panel with status badge, mode-pref radio, Home WiFi and AP password rows
- Changed: hostapd now runs as raspimidihub-hostapd.service (systemd) instead of subprocess.Popen + taskset + -B daemonise — cleaner stderr → journald capture under its own unit, automatic Restart=on-failure, CPU pin via CPUAffinity drop-in (still 0–2, leaves CPU 3 to the asyncio loop)
- Fix: AP service restart heals "no AP" / "two hostapds racing" states
- Fix: First hostapd spawn that loses the race against NetworkManager retries automatically
- Fix: Pi 3 WiFi power-save quirk — phones can re-associate cleanly after a client→AP cycle

System & performance
- Improved: CPU 3 reserved for the asyncio loop — idle CPU dropped from ~14 % to ~7 %, single-digit-ms loop lag
- Improved: SSE per-view subscription — each client only receives events its current view needs
- Improved: Heavy-controller load — 4 LaunchControl faders into Mixer 8 used to pin CPU at 100+ %; now ~80 % with single-digit-ms lag and zero SSE backlog
- Improved: SSE traffic cut roughly in half (clock dedup, plugin display throttle, ui_demo scope/meter halved)
- Improved: SSE connections clean up immediately when a tab closes (was waiting up to 30 s for the next heartbeat)
- Improved: System prepare on install disables services that contribute nothing to a single-purpose MIDI appliance (Bluetooth, ModemManager, cloud-init, cron, ntpsec / systemd-timesyncd, …)
- Improved: System prepare wraps service-disable + drop-in writes in a remount-rw / remount-ro pair — without it, `systemctl disable --now` was failing silently on the rosetup-managed read-only root because the SysV-init compat shim tries to update /etc/init.d/, leaving the unit "disabled" in our log but actually still running. Several service-disable entries (cloud-init, NTP, …) only actually take effect now
- Removed: rosetup deb's `Recommends: ntpsec` so a fresh install doesn't pull NTP in just to immediately disable it
- Added: Settings stats — Loop lag, MIDI in→out and Control in→MIDI out latency probes, process CPU %, SSE / sec, SSE backlog
- Added: Knob / wheel tick sounds toggle (Display section)

UI building blocks
- Added: Knob — round body with rotating indicator, LED arc, centre value text; adaptive drag sensitivity

- Improved: Buttons — same height as wheels (52 px) so mixed rows align
- Improved: Meter — read-only green-LED knob instead of a flat bar
- Improved: Cell config (Controller edit mode) — one card per cell with type / name / Ch / CC / Learn / On / Off
- Improved: Matrix clock indicator — pulses with the source's quarter notes
- Changed: Top-bar redesign — version + token stays in dim text, the old "stale, reload" inline link becomes a small pulsing red dot next to "RaspiMIDIHub" (tap = same hardReload), and a fullscreen toggle button sits to the right of "n devices" (uses the standard Fullscreen API, hides on iOS Safari which only supports it on <video>)

Packaging
- Added: CHANGELOG.txt installed at /usr/share/doc/raspimidihub/ inside the deb — same content as the GitHub release body, so an offline / SSH-only operator can `cat` the full release delta
- Changed: install.sh shipped with each GitHub release pins to that release's tag. Running an old install.sh installs the version it came from, not whatever's now /latest
- Changed: python3-dbus-next is a Recommends, not a hard Depends — Pis upgrading from 2.x via the legacy dpkg path install cleanly even when offline; BLE-MIDI silently disables until python3-dbus-next is installed (UI surfaces a one-tap Reinstall button that pulls the dep via the WiFi-dance). 3.x → 3.x+1 via the new apt-based install pulls Recommends by default
